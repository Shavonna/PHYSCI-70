Programming my jewlery shelf to rotate slowly.

For this week, I decided to improve upon my jewelry shelf by rotating it a fixed speed, determined first by the outputs of the potentiometer, and then by custom code that rotated the shelf in the following way.


First step: Learning how fast I want the jewlery stand to spin with the potentiometer

  • For the first step, I used my computer as an output devide and the motor's spinning speed was displayed depending on what value the potentiometer was at. This was recorded with the following code, which allowed me to create a new program with this approximate number in mind. By determining this spinning speed, this code allowed me to remove the potentiometer entriely from my setup and only use my written code to power my arduiono board and thus my jewlery shelf.

              pinMode(3, OUTPUT);
              pinMode(A0, OUTPUT);  //This will be GND for the potentiometer

              digitalWrite(A0, LOW);

              Serial.begin(9600);

            void loop() {
              int pot_value = analogRead(A2);   //pot wiper is on A2
              int motor_speed = map(pot_value, 0, 1023, 255, 0); // map motor_level to pot_value so that zero corresponds to lowest speed
              analogWrite(3, motor_speed);
              delay(1);
            
              // print out the value you read:
              Serial.println(pot_value);
            }

    For the second part, I wrote a code to then take the value we obtained from the potentiometer code, and use that as the new rotating speed of the shelf.

  • As I previously mentioned, the first code I wrote was only really to figure out what input I would need to power my jewerly shelf at the speed I desired. Once I obtained this speed, I then was able to put it into a new code where I controlled the speed of the shelf. After successfully figuring out this code, I then began to experiment with it, by adding buttons to control directoin, and some to act as on/off switches. The resulting final code uses buttons to control the direction, but the speed is fixed at the value's we were able to obtain from part one. The code is below.
